# ДАО-Дент: CRM-система для стоматологической клиники

## О проекте

Это проект CRM-системы и лендинга для стоматологической клиники «ДаоДент», расположенной у метро Семёновская (Москва). Проект построен на базе бойлерплейта CRM-boilerplate (Next.js 16, React 19, Prisma, PostgreSQL, NextAuth v5, shadcn/ui, Vercel AI SDK).

## Стек технологий (из бойлерплейта)

- **Frontend**: Next.js 16 (App Router), React 19, Tailwind CSS v4, shadcn/ui, Radix UI, Recharts
- **Backend**: Next.js API Routes, Prisma ORM, PostgreSQL 16
- **Auth**: NextAuth v5 (next-auth@5.0.0-beta.30)
- **AI**: Vercel AI SDK + DeepSeek / OpenAI
- **Video**: MUX (mux-node, mux-player-react)
- **Forms**: react-hook-form + zod
- **Деплой**: Docker (см. /deploy)

## Что нужно адаптировать из бойлерплейта

Бойлерплейт изначально создан для «Школа косметологов DIB Academy». Необходимо:

1. **Prisma-схема**: Переделать под стоматологическую клинику (пациенты вместо студентов, зубная формула, планы лечения, медкарты)
2. **Роли**: Владелец, Управляющий, Врач, Администратор (вместо ADMIN/STUDENT)
3. **Модули LMS → Медицина**: Заменить курсы/модули/уроки на приёмы/процедуры/планы лечения
4. **CRM**: Адаптировать Client/Lead/ClientNote под стоматологическую воронку
5. **Лендинг**: Полностью новый, с геопривязкой к м. Семёновская
6. **AI-агенты**: Голосовой ввод для врачей, распознавание фото, маркетинговый агент

## ТЕХНИЧЕСКОЕ ЗАДАНИЕ (полный текст)

### 1. Общие сведения о проекте

Настоящее техническое задание описывает требования к разработке комплексной цифровой платформы для стоматологической клиники «ДаоДент», расположенной в районе станции метро Семёновская (Москва). Платформа объединяет шесть функциональных модулей в единую экосистему, позволяющую автоматизировать все ключевые бизнес-процессы клиники — от привлечения пациентов до ведения медицинской документации.

**Цели проекта:**
- Создать лендинг с геопривязкой к м. Семёновская для привлечения пациентов из близлежащих районов (Соколиная Гора, Измайлово, Электрозаводская, Преображенское)
- Разработать полноценную CRM-систему с нуля, заточенную под специфику стоматологической клиники
- Внедрить AI-инструменты: голосовой ввод данных врачами, распознавание фотографий для автозаполнения карт
- Автоматизировать маркетинг: AI-агент для управления рекламой в Яндекс.Директ и публикации контента в соцсетях
- Обеспечить владельцу полный контроль: дашборд, аналитика, видеонаблюдение в реальном времени

**Инфраструктура клиники:**
- 3 стоматологических кабинета, в каждом — рабочая станция (компьютер)
- Рабочее место администратора на ресепшне
- Система видеонаблюдения (камеры в кабинетах, зоне ожидания, ресепшне)
### 2. Анализ рынка и конкурентной среды

На основе проведённого исследования (Perplexity Research, март 2026) выявлены следующие характеристики стоматологического рынка в районе м. Семёновская.

**2.1. Конкурентное окружение**

В зоне шаговой доступности от м. Семёновская функционирует более 20 частных стоматологических клиник. Диапазон цен — от 50-100 руб. за простые процедуры до 400-430 тыс. руб. за комплексную имплантацию и ортодонтию. Ключевые конкуренты:

| Клиника | Характеристика |
|---------|---------------|
| Ваша Стоматология | Семейный формат, средний сегмент, сильный локальный бренд |
| Макси-Дент | Широкий спектр услуг, активное SEO-продвижение |
| Микдент | Положительные отзывы на Яндекс.Картах, акцент на доверии |
| Master Dent | Премиум-позиционирование, открытый прайс-лист |

**2.2. Ключевые выводы из исследования**
- Пациенты ищут клиники по связке «услуга + метро» — это подтверждает необходимость жёсткой геопривязки в SEO
- Большинство клиник используют сайты-визитки; полноценная автоматизация с AI-инструментами даст серьёзное конкурентное преимущество
- Открытый прайс-лист — обязательный элемент; формы «узнать стоимость» отталкивают пациентов
- Реальные фото клиники, врачей и кабинетов значительно повышают доверие по сравнению со стоковыми изображениями
- Обязательные элементы лидогенерации: онлайн-запись, квизы, мессенджеры (WhatsApp/Telegram/Max), обратный звонок
### 3. Модуль 1: Лендинг клиники

Лендинг является основной точкой входа для потенциальных пациентов. Главная задача — конвертировать посетителей в заявки, с фокусом на жителей районов Соколиная Гора, Измайлово, Электрозаводская, Преображенское.

**3.1. Структура и контент лендинга**

Первый экран (hero-секция):
- Заголовок с геопривязкой: «Семейная стоматология у метро Семёновская — 5 минут пешком»
- Подзаголовок с УТП клиники (безболезненное лечение, современное оборудование, AI-технологии)
- CTA-кнопка «Записаться на приём» — фиксированная, видна при прокрутке (sticky)
- Кликабельный номер телефона в шапке
- Реальные фотографии клиники (не стоковые)

Обязательные секции лендинга:

| Секция | Описание |
|--------|----------|
| Услуги | Каталог услуг с карточками: терапия, хирургия, имплантация, ортодонтия, эстетика, детская стоматология. Каждая услуга — отдельная посадочная страница с описанием, ценами, фото до/после |
| Цены | Открытый прайс-лист таблицей. Пакеты услуг: «Комплекс лечения кариеса», «All-on-4 под ключ». Информация о рассрочке и кредите. Калькулятор стоимости (интерактивный) |
| Врачи | Карточки врачей: фото, стаж, специализация, сертификаты, образование. Кнопка «Записаться к этому врачу» на каждой карточке |
| Отзывы | Видео-отзывы пациентов, интеграция с отзывами Яндекс.Карт и Google. Галерея работ «До/После» с описанием случая |
| О клинике | Фотогалерея кабинетов и оборудования, 3D-тур (опционально). Лицензии, сертификаты, гарантии |
| Контакты | Интерактивная карта Яндекс/Google/2ГИС. Схема прохода от метро. Режим работы, парковка. Ссылки на мессенджеры |
| Акции | Блок с текущими акциями и специальными предложениями для новых пациентов. Скидки для жителей района |
**3.2. Геопривязка и локальное SEO**

Контентный уровень:
- Заголовки и мета-теги содержат «м. Семёновская», «Соколиная Гора», «Измайлово»
- Отдельная SEO-страница «Стоматология на Семёновской» с описанием маршрута от метро
- Блок «Почему жители Соколиной Горы / Измайлово выбирают нас»
- Адаптация текстов под локальные запросы: «стоматология семеновская», «зубной врач семеновская»

Технический уровень:
- Schema.org разметка: LocalBusiness + GeoCoordinates + DentalClinic
- Корректная карточка в Яндекс.Бизнес, Google My Business и 2ГИС
- Микроданные для расширенных сниппетов (рейтинг, часы работы, адрес)
- Региональная привязка в Яндекс.Вебмастер

**3.3. Система сбора заявок**

Формы захвата лидов:
- Форма онлайн-записи: выбор услуги, врача, даты и времени (интеграция с расписанием CRM)
- Квиз-опрос: «Подберите план лечения» / «Узнайте, подойдут ли вам элайнеры» — результат на email + создание лида в CRM
- Виджет обратного звонка: «Перезвоним за 30 секунд»
- Чат-виджет с интеграцией WhatsApp, Telegram и Max
- Поп-ап при попытке ухода: спецпредложение для первого визита

**Важно:** Все формы должны автоматически создавать записи в CRM-системе с фиксацией источника трафика (UTM-метки), времени обращения и типа запроса. Каждая заявка получает уникальный ID и попадает в воронку обработки.

**AI-ассистент на главной странице:**

На первом экране лендинга размещается интерактивный AI-ассистент (чат-бот), который в режиме реального времени:
- Отвечает на вопросы посетителей об услугах, ценах, врачах и подготовке к процедурам
- Помогает подобрать удобное время для записи, показывая свободные слоты в расписании врачей с учётом выбранной услуги
- Автоматически создаёт заявку в CRM и отправляет подтверждение пациенту в мессенджер (WhatsApp/Telegram/Max) или по SMS
- Работает 24/7, обеспечивая захват лидов в нерабочее время клиники
### 4. Модуль 2: Панель владельца клиники

Панель владельца — центральный командный пункт, предоставляющий полную картину состояния бизнеса в реальном времени. Доступ по ролевой модели — только для владельца и управляющего.

**4.1. Дашборд руководителя**

Показатели в реальном времени:
- Количество пациентов на приёме прямо сейчас (по каждому из 3 кабинетов)
- Очередь записанных пациентов на сегодня / неделю
- Статус каждого кабинета: свободен / приём / перерыв / стерилизация
- Количество новых заявок с сайта за сегодня / неделю / месяц
- Конверсия заявок: заявка → запись → визит → оплата

Управленческие отчёты:
- Загруженность врачей: процент занятости по дням, неделям
- Средний чек по врачам и услугам
- Показатель возвращаемости пациентов
- Эффективность рекламных каналов: стоимость привлечения пациента (CAC) по каждому каналу
- Рейтинг врачей по отзывам и NPS

**4.2. Финансовая аналитика**
- Выручка за период (день/неделя/месяц/квартал/год) с разбивкой по услугам
- Динамика доходов — графики и сравнение с предыдущими периодами
- Задолженность пациентов (неоплаченные планы лечения)
- Расходы клиники: зарплаты, материалы, аренда, реклама
- Прибыль / рентабельность по услугам и врачам

**4.3. Уведомления и алерты**
- Уведомление при поступлении новой заявки с сайта
- Алерт, если кабинет простаивает больше установленного времени
- Оповещение о негативном отзыве на площадках
- Уведомление о критически низком остатке материалов на складе
- Push-уведомления на телефон владельца (мобильная версия или Telegram-бот)
### 5. Модуль 3: Кабинет врача

Кабинет врача — рабочий интерфейс для стоматологов. Установлен на компьютере в каждом из трёх кабинетов клиники. Ключевое требование — минимум ручного ввода, максимум автоматизации через голосовые и визуальные AI-инструменты.

**5.1. Расписание и приёмы**

Рабочий экран врача:
- Расписание на день: список пациентов с временем приёма, типом услуги, краткой историей
- Карточка текущего пациента: ФИО, возраст, аллергии, хронические заболевания, история посещений
- Зубная формула (интерактивная): визуальная схема зубов с возможностью отметки состояния каждого зуба
- План лечения: текущий и предложенный, с расчётом стоимости
- Быстрая запись следующего визита прямо из карточки приёма

Типы приёмов:

| Тип приёма | Описание |
|------------|----------|
| Первичная консультация | Осмотр, диагностика, составление плана лечения. Включает фотопротокол |
| Лечебный приём | Выполнение процедур по плану лечения. Фиксация выполненных работ |
| Контрольный визит | Проверка результатов лечения, коррекция плана |
| Экстренный приём | Острая боль, травма. Приоритетная запись вне очереди |
| Профилактика | Гигиена, профосмотр, снятие камня |

**5.2. Голосовой ввод информации (AI-агент)**

Одна из ключевых инноваций системы — голосовой ввод данных. Врач во время приёма диктует информацию, AI-агент в реальном времени распознаёт речь и заполняет соответствующие поля в электронной медицинской карте.

Функциональные требования:
- Распознавание русской речи в реальном времени с точностью не менее 95%
- Знание стоматологической терминологии: названия зубов (по международной нумерации), процедур, материалов, диагнозов
- Автоматическое заполнение полей карточки приёма: диагноз, выполненные процедуры, использованные материалы, рекомендации- Команды голосом: «Начать приём», «Закрыть приём», «Добавить диагноз», «Назначить следующий визит через 2 недели»
- Подтверждение врачом: после диктовки система показывает распознанный текст, врач подтверждает или корректирует

Сценарий работы:
1. Врач нажимает кнопку «Начать запись» или говорит голосовую команду
2. AI-агент активирует микрофон и начинает распознавание
3. Врач диктует: «Пациент Иванов, осмотр, зуб 36, диагноз — средний кариес, проведено лечение, пломба светоотверждаемая»
4. Система автоматически заполняет: поле «Зуб» — 36, «Диагноз» — средний кариес, «Процедура» — лечение кариеса, «Материал» — светоотверждаемая пломба
5. Врач визуально проверяет и подтверждает одним нажатием

**5.3. Распознавание фотографий (AI-агент)**

⚠ Требует обсуждения — AI-инструменты второй очереди. Врач или ассистент делают фото клинической картины, рентгеновский снимок или документ, и система автоматически извлекает данные и вносит их в карту.

Возможности:
- Загрузка фото через камеру компьютера, телефон (через веб-интерфейс) или сканер
- Автоматическое прикрепление фото к карточке пациента с датой и описанием
- AI-анализ рентгеновских снимков: подсвечивание проблемных областей (кариозные полости, воспаления, переломы корней)
- Фотопротокол приёма: серия фото «до» и «после» с автоматической привязкой к записи приёма
- Распознавание документов: паспорт, полис ОМС/ДМС — автозаполнение данных пациента
### 6. Модуль 4: CRM-система

Центральное ядро всей платформы. CRM объединяет данные из всех модулей и является единым хранилищем информации о пациентах, записях, оплатах и коммуникациях. Разрабатывается полностью с нуля под специфику клиники ДАО-Дент.

**6.1. Карточка пациента**

Основные данные:
- ФИО, дата рождения, контактные данные (телефон, email, мессенджеры)
- Адрес (для понимания географии пациентов)
- Источник привлечения: с какого канала пришёл (SEO, контекст, рекомендация, соцсети)
- Статус в воронке: новый лид / записан / был на приёме / на лечении / завершил лечение / потерянный
- Документы: паспорт, полис, согласия на обработку данных и медицинское вмешательство

Медицинские данные:
- Аллергии, хронические заболевания, противопоказания
- Зубная формула с историей изменений
- Все планы лечения (текущие и архивные)
- Фотопротоколы и рентгеновские снимки
- Записи приёмов с голосовыми протоколами

Финансовые данные:
- История оплат, задолженности
- Активные рассрочки / кредиты
- Сумма затрат пациента за всё время (LTV)

**6.2. Воронка продаж и коммуникации**

| Этап | Описание | SLA |
|------|----------|-----|
| Новая заявка | Заявка поступила с сайта, звонка или мессенджера. Автоматически назначается ответственный администратор | до 5 мин |
| Первый контакт | Администратор связался с пациентом, выяснил потребность, предложил услугу | до 15 мин || Записан на приём | Пациент подтвердил запись. Автоматическое SMS/WhatsApp-напоминание за 24ч и за 2ч | авто |
| Пришёл на приём | Пациент явился. Статус обновляется автоматически при отметке в системе | авто |
| План лечения | Врач составил план. Пациенту отправляется план лечения с расчётом стоимости на email | до 24ч |
| На лечении | Пациент проходит курс лечения. Система отслеживает выполнение плана | авто |
| Завершил лечение | Лечение завершено. Автоматический запрос отзыва. Напоминание о профосмотре через 6 мес | авто |

Автоматические коммуникации:
- SMS / WhatsApp / Telegram / Max — напоминания о записи (за 24ч, за 2ч) — там, где удобно клиенту
- Поздравления с днём рождения + спецпредложение
- Напоминание о профосмотре каждые 6 месяцев
- Запрос отзыва после завершения лечения
- Реактивация «потерянных» пациентов: серия писем через 1, 3, 6 месяцев после последнего визита

**6.3. Электронная медицинская карта (ЭМК)**
- Соответствие требованиям Минздрава РФ к ведению медицинской документации
- Формирование информированных согласий на медицинское вмешательство (шаблоны с автозаполнением)
- Печать выписок, справок, направлений
- Возможность интеграции с ЕГИСЗ (передача данных в единую систему)
- Хранение всех фото, снимков и документов в привязке к пациенту
### 7. Модуль 5: AI-агент маркетинга

Отдельный интеллектуальный модуль, автоматизирующий маркетинговую деятельность клиники. Состоит из двух подсистем: управление контекстной рекламой и управление контентом в соцсетях.

**7.1. Управление Яндекс.Директ**

Интеграция с API Яндекс.Директ:
- Подключение через Яндекс.Директ API v5 для полного управления рекламными кампаниями
- Создание и управление кампаниями: поисковые, РСЯ (рекламная сеть Яндекса), ретаргетинг
- Автоматическая настройка геотаргетинга: м. Семёновская, радиус 3-5 км, районы Соколиная Гора, Измайлово, Преображенское

Функции AI-агента:
- Автоматический подбор и группировка ключевых слов по услугам клиники
- Генерация объявлений: заголовки, тексты, быстрые ссылки — на основе услуг и акций клиники
- Оптимизация ставок: AI анализирует конверсии из CRM (заявка → визит → оплата) и автоматически корректирует ставки
- A/B-тестирование объявлений: автоматическая ротация вариантов, отключение неэффективных
- Отчёты владельцу: расход бюджета, количество кликов, заявок, стоимость привлечения пациента (CPL)

Дашборд рекламы (в панели владельца):
- Текущий расход бюджета за день / неделю / месяц
- Количество показов, кликов, заявок с разбивкой по кампаниям
- ROI по каждой кампании: сколько потрачено → сколько выручки принесли пациенты с этого канала
- Рекомендации AI: «Увеличить бюджет на имплантацию — высокая конверсия» или «Остановить кампанию по отбеливанию — низкий ROI»
**7.2. Контент-менеджер соцсетей**

Поддерживаемые платформы:
- ВКонтакте (основная площадка)
- Telegram-канал клиники
- YouTube / RuTube (видеоконтент, длинные ролики)
- Яндекс.Дзен (статьи, нативный контент, SEO-эффект)
- Одноклассники (опционально, для старшей аудитории)
- TikTok (короткие видео: лайфхаки, «до/после», закулисье клиники)
- ВК Видео (дублирование видеоконтента из TikTok / YouTube)
- Instagram* (Stories, Reels — адаптация контента для визуальной аудитории)
- Мессенджер Max (канал клиники, рассылки акций и напоминаний)

*Meta признана экстремистской организацией на территории РФ

Функции AI-агента:
- Генерация контент-плана на месяц: темы постов, типы контента, расписание публикаций
- Создание текстов постов: AI формирует текст на основе услуг клиники, акций, советов по уходу за зубами
- Подготовка визуалов: подбор и адаптация фото/видео из фотопротоколов клиники (с согласия пациентов)
- Автоматическая публикация по расписанию из контент-плана через API соцсетей
- Мониторинг комментариев и упоминаний: уведомление администратора при негативных комментариях
- Аналитика вовлечённости: охваты, лайки, комментарии, переходы на сайт

Видеопроизводство:
- Организация съёмки видеороликов непосредственно в клинике
- Съёмка процесса лечения (с согласия пациента) — формат «до/после»
- Интервью с врачами: экспертные советы, ответы на частые вопросы
- Обзоры оборудования и технологий клиники
- Советы по уходу за зубами — короткие обучающие ролики
- Адаптация контента под форматы: TikTok, YouTube Shorts, Reels, ВК Видео
**7.3. Управление отзывами**

Репутация клиники в поисковых системах и на картах — ключевой фактор привлечения новых пациентов. AI-агент автоматизирует работу с отзывами.

Платформы мониторинга:
- Яндекс.Карты — мониторинг новых отзывов, уведомления, генерация ответов
- 2ГИС — отслеживание рейтинга, оперативная реакция на негатив
- Google Maps — контроль отзывов для англоязычной аудитории

Функции:
- Автоматическое уведомление при появлении нового отзыва (push + Telegram)
- AI-генерация персонализированных ответов на положительные отзывы
- Эскалация негативных отзывов: алерт владельцу + шаблон ответа
- Автоматический запрос отзыва у пациента после завершения лечения
- Дашборд: динамика рейтинга, количество отзывов по площадкам
### 8. Модуль 6: Видеонаблюдение

Модуль интеграции системы видеонаблюдения клиники с панелью владельца. Владелец может просматривать видеопотоки со всех камер в реальном времени через веб-интерфейс панели управления.

Функциональные требования:
- Просмотр live-видео со всех камер клиники в панели владельца
- Мультикамерный режим: отображение всех камер одновременно на одном экране
- Возможность выбора конкретной камеры для полноэкранного просмотра
- Архив записей: просмотр записей за прошлые дни (глубина хранения — настраивается)
- Доступ с мобильных устройств: адаптивный интерфейс для смартфона/планшета

Камеры и зоны:

| Зона | Описание камеры |
|------|----------------|
| Кабинет 1 | Обзорная камера рабочей зоны врача |
| Кабинет 2 | Обзорная камера рабочей зоны врача |
| Кабинет 3 | Обзорная камера рабочей зоны врача |
| Ресепшн | Зона приёма и ожидания пациентов |
| Зона ожидания | Общий зал ожидания |
| Вход | Входная группа клиники |

**Важно:** Видеонаблюдение в медицинских кабинетах должно соответствовать требованиям ФЗ-152 «О персональных данных» и ФЗ-323 «Об основах охраны здоровья граждан». Необходимо информированное согласие пациентов на видеозапись.
### 9. Требования к безопасности и законодательству

**9.1. Защита персональных данных**
- Соответствие ФЗ-152 «О персональных данных»: согласия, обработка, хранение, удаление
- Шифрование данных при передаче (HTTPS/TLS) и хранении (AES-256)
- Двухфакторная аутентификация для всех пользователей системы
- Ролевая модель доступа: владелец, управляющий, врач, администратор — каждый видит только свои данные
- Журнал действий (audit log): кто, когда, какие данные просматривал/изменял
- Резервное копирование данных: ежедневное автоматическое, хранение не менее 90 дней

**9.2. Медицинское законодательство**
- Соответствие ФЗ-323 «Об основах охраны здоровья граждан в РФ»
- Ведение медицинской документации по стандартам Минздрава
- Возможность формирования документов для проверяющих органов
- Хранение медицинских данных на территории РФ (ФЗ-242)
- Готовность к интеграции с ЕГИСЗ

**9.3. Рекламное законодательство**
- Соблюдение ФЗ-38 «О рекламе» в части рекламы медицинских услуг
- Обязательная пометка «Имеются противопоказания. Необходима консультация специалиста»
- Маркировка рекламы по ФЗ-347 (единый реестр интернет-рекламы, ОРД)
- AI-агент маркетинга должен автоматически добавлять обязательные дисклеймеры
