# Интеграция оплаты — выдача доступа автоматически

Сейчас доступ выдаёт **админ** вручную через «Управление пользователями» → «Выдать доступ».  
После подключения оплаты доступ можно выдавать **автоматически** после успешной оплаты.

## Как устроено сейчас

- **Создание доступа** = создание/обновление пользователя (User) + запись на курсы (Enrollment).
- API: `POST /api/admin/users` с телом `{ email, name?, courseIds[] }`.
  - Если пользователя нет — создаётся с сгенерированным паролем и записью на курсы.
  - Если пользователь есть — добавляются записи на выбранные курсы (без смены пароля).

## Как подключить оплату

1. **Внешняя платёжная система** (ЮKassa, Stripe, Tinkoff и т.д.) после успешной оплаты вызывает ваш **webhook** или вы опрашиваете статус платежа.

2. **API для выдачи доступа по оплате уже добавлен**:
   - `POST /api/payment/grant-access`
   - Header: `x-payment-secret: <PAYMENT_WEBHOOK_SECRET>`
   - Тело: `{ email, name?, courseId | courseIds[], licenseDays?, paymentId? }`
   - Логика:
     - найти или создать пользователя по `email`;
     - выдать доступ к курсам;
     - если доступ был отозван или истёк — восстановить;
     - отправить email пользователю (если настроены `RESEND_API_KEY` и `EMAIL_FROM`).

3. **Переиспользуемая логика выдачи доступа вынесена**:
   - `lib/access.ts` (`grantCourseAccess`) — используется и админкой, и payment webhook.

4. **Письмо пользователю**: после выдачи доступа отправить email с ссылкой на вход и паролем (или ссылкой для установки пароля), например через Resend, SendGrid, Nodemailer.

## Пример сценария с оплатой

```
Пользователь оплатил курс на сайте → платёжка вызывает webhook →
ваш API проверяет подпись → создаёт User (если нет) + Enrollment →
отправляет email с логином и паролем (или ссылкой на сброс пароля).
```

Рекомендуется вынести общую логику «выдать доступ» (создание User + Enrollment) в отдельную функцию (например в `lib/access.ts`) и вызывать её и из `POST /api/admin/users`, и из обработчика оплаты.
